TRABAJO PRÁCTICO Nº 2: Drivers

Objetivos de aprendizaje
  • Conocer los conceptos básicos de drivers.
  • Adquirir habilidad en la instalación y actualización de drivers
Unidad temática que incluye este trabajo práctico

Este trabajo práctico corresponde a la unidad Nº:1 de la programación de la asignatura.

Consignas a desarrollar en el trabajo práctico

Drivers. Seguro que has leído esa palabra un montón de veces. Pero ¿qué son?. Sin drivers no podrías jugar, imprimir documentos o reproducir música. “Pero eso lo hace el hardware”. Sí, pero el hardware no se controla solo: necesita órdenes.

Esas órdenes las da el sistema operativo, que las envía en un idioma abstracto, que los dispositivos no entienden. Para que cada aparato sepa qué se le pide, hacen falta programas que traducen cada comando del SO. Esos programas son los drivers.

El teórico puede ser consultado en el libro “Instalación y configuración de sistemas operativos” de Enrique Bellido Quintero, Unidad 2, Tema 4.

Preguntas
  1. ¿Qué es un driver o controlador de dispositivo? ¿Para qué sirve?

    2. Es un programa que permite al SO interaccionar con un periférico (impresora, scaner, etc) o dispositivo hardware (tarjeta gráfica, tarjeta de sonido, modem, etc), haciendo una abstracción del mismo y proporcionando una interfaz para poder utilizarlo

    Le indica al SO como controlar y comunicarse con un dispositivo particular.

    Tipos de Drivers

    • Existen tantos tipos de controladores como tipos de periféricos
    • Puede haber más de un controlador para un mismo dispositivo

    Ejemplos de Drivers:

    • de Audio (Realtek, SoundBlast)
    • de Video (ATI, NVIDIA)
    • LAN o Ethernet
    • Wireless
    • Escáners
    • Impresoras
    • Mouse
    • Teclados
    • Webcam

    Tipos de drivers:

    • Fabricantes de dispositivos
    • Genéricos
    • Comunidad de Usuarios
  2. ¿Por qué es recomendable mantener los drivers actualizados?

    3. Los fabricantes de hardware suelen actualizar los drivers para otorgar más funcionalidades, mejorar el rendimiento o aumentar las medidas de seguridad con la finalidad de corregir errores o vulnerabilidades que podrían comprometer el equipo de los usuarios.

  3. ¿Qué son los dispositivos PnP o “plug and play”?

    4. En español "enchufar, conectar y usar" es la tecnología o cualquier avance que permite a un dispositivo informático ser conectado a una computadora sin tener que configurar, mediante jumpers o software específico (no controladores) proporcionado por el fabricante, ni proporcionar parámetros a sus controladores. Para que sea posible, el sistema operativo con el que funciona el ordenador debe tener soporte para dicho dispositivo.

  4. En Windows:
    1. ¿Qué es el administrador de dispositivos?

      2. Es una utilidad de Windows a través de la cual podemos gestionar todo el Hardware instalado en el equipo. Para abrirlo, abrimos el Panel de control, desde le menú Inicio.

    2. ¿Qué puede hacer el usuario a través del Administrador de Dispositivos?

      3. El usuario puede realizar lo siguiente:

      • Comprobar si el funcionamiento de los dispositivos es correcto
      • Actualizar el software de controlador o Driver
      • Modificar o establecer propiedades a un dispositivo
      • Habilitar o deshabilitar dispositivos
    3. ¿Cómo se distribuyen los Drivers en este sistema operativo?

      4. Una vez abierto el Administrador de dispositivos, se mostrara una lista con todos los dispositivos instalados en el equipo de manera agrupada. Para ver el contenido de cada uno hacer clic sobre el signo que hay a la izquierda de cualquiera de los grupos de dispositivos.

      En caso de que un dispositivo no funcione correctamente o hubiese algún error, el dispositivo se mostrara con un signo de interrogación amarillo.

      Simbolo Definición
      Ocurrió un problema. Muestra un código de error del dispositivo.
      El dispositivo está desactivado (está presente físicamente, consume recursos, pero no tiene un controlador cargado en modo protegido.
      Configuración manual en lugar de automática.
      El controlador específico del dispositivo no está disponible y se instalo un controlador compatible.

    2. ¿Cómo se llama a los drivers en linux?

      3. Los drivers son módulos del Kernel

      El kernel de Linux tiene un diseño modular. En el momento de arranque, sólo se carga un kernel residente mínimo en memoria. Por ello, cuando un usuario solicita alguna característica que no esta presente en el kernel residente, se carga dinámicamente en memoria un módulo kernel, también conocido algunas veces como un controlador.
