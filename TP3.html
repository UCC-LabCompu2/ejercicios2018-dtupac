Objetivos de aprendizaje
  • Adquirir conceptos básicos sobre simulación de circuitos eléctricos.
  • Utilizar herramientas que permitan la simulación de circuitos.
Unidad temática que incluye este trabajo práctico

Este trabajo práctico corresponde a la unidad Nº: 7 de la programación de la asignatura.

Preguntas

Ejercicios
  1. Desde la consola de comandos, resuelva el siguiente ejercicio:

    2. Proyecto local

    Nº Acción Comando en Git
    1 Inicializar un repositorio, Crear un archivo y guardarlo git init
    2 Mostrar situación actual del repositorio git status
    3 Registrar los cambios (añadir el archivo) git add archivo_nuevo.txt
    4 Incluir el archivo en el repositorio local git commit -m "Incluyo el archivo al repositorio local"
    5 Mostrar los commits realizados git log
    6 Modificar nuevamente el archivo git status
    7 Registrar los cambios e incluirlos git commit -a -m "Registro cambio y los incluyo"
    8 Mostrar los commits realizados git log

  2. Cree una cuenta de Github, entrando a https://github.com/
  3. Empleando Github:
    1. Crear un repositorio compartido sin inicializarlo

    2. Configurar la información de usuario

      3. https://github.com/dtupac/Ejercicio3_TPN3

    3. Clonar el repositorio
    4. Agregar un archivo y comitearlo de manera local
    5. Guardar el commit en el repositorio remoto
    6. En el navegador, visualizar los commits realizados
  4. Empleando GitHub (si utiliza comandos de git, especifique cuales son):
    1. Crear un repositorio e Inicializarlo. Agregar una descripción
    2. Configurar el repositorio para trabajar de manera colaborativa con un compañero. (Setting>Collaborators).
    3. Clonar el repositorio. Ver en 2.c)
    4. Crear un archivo dentro del repositorio.
    5. Verificar el estado del repositorio. De ser necesario, agregar el archivo y guardar los cambios de forma local.
    6. Guardar los archivos en el repositorio remoto (GitHub).

    7. Si el compañero ha realizado un cambio (si está trabajando de manera individual, realizar un cambio desde el navegador, y realizar un commit desde el mismo), actualizar el repositorio local.
  5. Empleando el repositorio del ejercicio 3:
    1. Cree una nueva rama

      2. git branch nueva_rama

    2. Vea el listado de todas las branchs existentes en el proyecto y la branch actual

      3. git branch

    3. Cambie de rama

      4. git checkout nueva_rama

      A continuación, todos los comandos y sus respectivas salidas:

    4. Realice algún cambio en el código y push al repositorio remoto
    5. Visualice en Github los commits de las branchs


    6. Merge de la nueva rama con Master

      7. Estando en Master se debe ejecutar el siguiente comando: git merge nueva_rama

      Salida:

    7. Borrar una branch

  6. Empleando el repositorio del ejercicio 1:
    1. Volver a un commit anterior (sin borrar el posterior o último)

      2. Paso 1: Listamos todos los commits

      git log

      Paso 2: Elijo cualquier commit y lo uso con el comando:

      git checkout “id_commit”

    2. Volver al último commit

      3. En esta ocasión usamos el comando:

      git log -f

      O si usamos bash, podemos usar el comando:

      git log -g | grep 'commit '

      Luego con la lista de commits podemos movernos en cualquiera. Y si queremos volver al último:

      git checkout master

    3. Descartar el ultimo commit (usado reset hard).

  7. Empleando el repositorio del ejercicio 3:
    1. Eliminar un archivo: Primero listamos los archivos en el directorio con git ls-files luego eliminamos el archivo elegido con git rm [archivo_a_borrar]
    2. Eliminar una carpeta: uso el comando git rm -r [nom_dir_a_borrar]
    3. Guardar los cambios en repositorio remoto.
    4. Verificar que en el remoto, los archivos no estén presentes.
  8. Desde GitHub realizar un Fork de las filminas usadas en clases. https://github.com/UCCLabCompu2/filminas.git
  9. Configure WebStorm para clonar el repositorio al que se le realizó el Fork.
