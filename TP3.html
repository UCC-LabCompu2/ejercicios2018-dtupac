Objetivos de aprendizaje

  • Adquirir experiencia en la utilización de sistemas de gestión de versiones
Unidad temática que incluye este trabajo práctico

Este trabajo práctico corresponde a la unidad Nº:2 de la programación de la asignatura.

Consignas a desarrollar en el trabajo práctico

Existen diversas herramientas que nos ayudan en la tarea de desarrollar software y hacen a las buenas prácticas del programador.

La disponibilidad de herramientas para desarrolladores es muy amplia. Hay unas más completas y complejas que otras, todo dependiendo de las necesidades individuales o de conjunto de una organización. Es en ese sentido que han surgido varias herramientas para desarrollo en equipos con el fin de organizar y facilitar un poco su flujo de trabajo.

Preguntas
  1. ¿Qué son las herramientas de desarrollo? ¿Qué tipo de herramientas existen?

    2. Son herramientas que permiten agilizar el proceso de desarrollo de Software.

    Son aplicaciones o programas que puede contar con multiples herramientas que se encarguen de funciones especificas y afines para realizar una tarea.

    Existen las siguientes tipos de herramientas:

    • IDE
    • Sistema de Control de Versiones
    • Bug Tracker
    • Pruebas de desempeño, carga, stress
    • Documentación
  2. ¿Qué significa IDE? De ejemplos de IDE’s para desarrollo web ampliamente usados.

    3. IDE en español significa AMBIENTE DE DESARROLLO INTEGRADO: Es un software que proporciona servicios integrales para facilitarle al programador el desarrollo de software.

    Ejemplos de IDE's para desarrollo Web

    • Eclipse
    • Net Beans
    • JetBrains Web Storm
    • Aptana
    • Dreamweaber
    • Microsoft Visual Studio Express for Web
    • Sublime Text
    • NotePad ++
  3. ¿Qué herramientas suele integrar un entorno de programación? Describa cada una.

    4. Un entorno de programación suele integrar las siguientes herramientas:

    • Editor de código fuente:

      • Es un procesador de textos orientado para escribir código fuente de aplicaciones en general en lenguajes de programación.

    • Herramientas de construcción automáticas:

      • Generalmente los editores de código soportan varios lenguajes y son capaces de abrir varios archivos a la vez, resaltar su sintaxis y ofrecer ayudas contextuales a la hora de escribir o visualizar el código de las aplicaciones.

    • Depurador o Debugger:

      • Permite probar y eliminar los errores del programa.

      El depurador permite detener el programa en un punto determinado o en un momento determinado para que el usuario pueda examinar y modificar la memoria y las variables del programa, cambiar punto de ejecución, ejecutar una instrucción o partes determinadas de código.

    • Compilador:

      • Traduce del lenguaje de programación al lenguaje maquina, código intermedio o texto.

      • Lenguajes compilados: C, C++, Fortran, Pascal, Visual Basic.
      • Lenguajes interpretados: Python, JavaScript, HTML, Ruby, PHP.
    • Intérprete:

      • Realiza la traducción a medida que sea necesaria, instrucción por instrucción y no guardan el resultado de la traducción.

      Los programas interpretados suelen ser más lentos que los compilados debido a la necesidad de traducir el programa mientras se ejecuta, pero son más flexibles permitiendo reemplazar partes del programa.

      Uno de los entornos más comunes de uso de los intérpretes es en los navegadores web, debido a que se ejecutan independientemente de la plataforma.

  4. ¿Qué es un sistema de gestión de versiones o VCS? ¿Qué ventajas tiene?

    5. Permiten gestionar los archivos de un proyecto (y sus versiones) y que sus integrantes puedan acceder remotamente a ellos.

    Con los VCS podemos:

    • Descargar los archivos.
    • Modificar archivos.
    • Publicar cambios.
    • Volver a versiones anteriores.
    • Ramificar a partir de una versión.
    • Aplicar cambios de distintas versiones.

    Ventajas:

    • Permite llevar cuenta de los cambios de un conjunto de archivos digitales en el tiempo
    • Cada versión registrada en el tiempo es una revisión
    • Permite tanto a un desarrollador como a un grupo de desarrolladores, gestionar el código del proyecto.

    Alguna de las VCS mas conocidas:

    • Git
    • SVN
    • Mercurial
    • Bazaar
  5. ¿Qué es Git? ¿Qué servidores se pueden utilizar para ello?

    6. Es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia y la confiabilidad del mantenimiento de versiones de aplicaciones cuando éstas tienen un gran número de archivos de código fuente. Su propósito es llevar registro de los cambios en archivos de computadora y coordinar el trabajo que varias personas realizan sobre archivos compartidos.

    Git: Servidores

    • Github
    • GitLab
    • Bitbucket
    • GNU Savannah
    • SourceForge
    • Gitea
    • Phabricator
    • Gogs
  6. ¿Por qué es importante la documentación de Código?

    7. La documentación de código es importante porque ayuda a entender lo que se esta haciendo y porque, y permite el mantenimiento del código.

    Niveles de documentación:

    • Por clase: descripción,autor,fecha y ultima modificación.
    • Por método: descripción del objeto, funcionalidades, parámetros y resultados.
    • De variables: que sean importantes o difíciles de entender.
    • De limitaciones.
    • De algoritmos implementados.

    Un Generador de Documentación: es una herramienta de programación que genera documentación destinada a los programadores (documentación de API) o a usuarios finales, o a ambos, a partir de un conjunto de código fuente especialmente documentado, y en algunos casos, archivos binarios.

  7. ¿Qué es doxygen? ¿Qué otras herramientas similares existen?

    8. Doxygen es un acrónimo de dox(document) gen(generator), generador de documentación para código fuente.

    Es un generador de documentación para C++, C, Java, Objective-C, Python, IDL (versiones Corba y Microsoft), VHDL y en cierta medida para PHP, C# y D. Dado que es fácilmente adaptable, funciona en la mayoría de sistemas Unix así como en Windows y Mac OS X. La mayor parte del código de Doxygen está escrita por Dimitri van Heesch.

    Herramientas Similares:

    • Docco
    • JSDoc
    • Doxx
    • Jdiff

    11. Para asegurar una buena calidad en los proyectos de dimension considerable es importante utilizar un Bug Tracker (BTS) para realizar el seguimiento de los defectos o Bugs que surgen y son detectados.

  11. ¿Qué son las pruebas de desempeño? ¿Qué tipo de pruebas se pueden realizar y para qué sirve cada una?

    12. Las Pruebas de Desempeño o Performance son pruebas que se realizan al software para probar ciertas cualidades del mismo.

    • Load Test (Pruebas de Carga): Simular la realidad a la que esta expuesto el sistema cuando este en produccion.
    • Stress Test (Pruebas de Strees): Simula mas carga de la esperada.
    • Endurance o Resistencia: Desempeño del sistema después de una carga duradera por un periodo de tiempo largo.
  12. ¿Qué es la integración continua? ¿Qué herramientas se pueden emplear para ello?

    13. Es una buena practica generar tests para verificar el correcto funcionamiento de las partes del software.

    Estos test pueden ser manuales (los realiza una persona según un documento) o automáticos (código que prueba otro código).

    En el caso de los test automatizados, se recurre muchas veces a servidores que corran los test, para poder correrlos en distintos ambientes. A estos servidores se les llama de Integración Continua.
