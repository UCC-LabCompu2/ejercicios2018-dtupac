Unidad temática que incluye este trabajo práctico

Este trabajo práctico corresponde a la unidad Nº: 4 de la programación de la asignatura.

Consignas a desarrollar en el trabajo práctico

La red de redes no sería lo que es hoy sin la ayuda de HTML5. HTML5 es un conjunto de lenguajes que posibilitan tener aplicaciones Web con grandes funcionalidades en nuestros navegadores.

La nueva versión de 5 de HTML nos da la libertad como programadores de realizar aplicaciones completas, juegos, animaciones, o simples artículos de Blog. En la actualidad, HTML5 no se limita solamente a navegadores web, sino que se pueden realizar aplicaciones móviles y de escritorio sin necesidad de estar conectados.

Preguntas
  1. ¿Qué es una aplicación web? De ejemplos de aplicaciones web.

    2. Es un software almacenado en una computadora o servidor web, que los usuarios pueden utilizar a través de internet o la intranet con un navegador web para obtener distintos servicios. Por ejemplo, Wikis, juegos, herramientas de diseño, mails, etc.

    • Web mails
    • Wikis
    • Tiendas en Línea
    • Ofimática
    • Dibujo
    • Juegos Online
    • Otros
      (Sistemas de Gestión, Diseño, Cálculo, etc)
  2. ¿Qué ventajas tienen las aplicaciones web? ¿Cómo se clasifican según el tipo de acceso?

    3. Las ventajas que tienen las aplicaciones web son:

    • Facilidad de acceso
    • Independencia de sistema operativo
    • Facilidad de actualización y mantenimiento
    • Bajo consumo de recursos
    • Utilizable desde cualquier plataforma

    Clasificación segun el tipo de acceso:

    • Públicas:Tiendas virtuales, diarios digitales, portales de internet, etc.
    • Restringidas:Intranets, que ofrecen servicios para mejorar gestiones internas de una empresa (control hs de personal, gestión ed proyectos o tareas), etc.
  3. ¿Cómo es la estructura típica de una aplicación web? ¿Cuántas capas posee y que función cumple cada capa?

    4. Normalmente se posee 3 capas.

    • Capa del Cliente
    • Capa Intermedia
    • Capa de Servidor

    11. HTML:

                        
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Texto en 2 Columnas</title>
    <link rel="stylesheet" type="text/css" href="estilo_2_columnas.css">
</head>
<body>
    <div class="wrap">
        <h2>C++ para Todos  y Todas</h2>
        <div class="sidebar">
            <p id="columna1">Texto columna 1</p>
        </div>

        <div class="main">
            <p>Al usar la directiva #include se le dice al compilador que busque e interprete todos los elementos definidos
                en el archivo que acompaña la directiva (en este caso, iostream). Para evitar sobrescribir los elementos
                ya definidos al ponerles igual nombre, se crearon los espacios de nombres o namespace del singular en inglés.
                En este caso hay un espacio de nombres llamado std, que es donde se incluyen las definiciones de todas
                las funciones y clases que conforman la biblioteca estándar de C++. Al incluir la sentencia using namespace
                std le estamos diciendo al compilador que usaremos el espacio de nombres std por lo que no tendremos
                que incluirlo cuando usemos elementos de este espacio de nombres, como pueden ser los objetos cout y cin,
                que representan el flujo de salida estándar (típicamente la pantalla o una ventana de texto) y el flujo
                de entrada estándar (típicamente el teclado).</p>

            <p>La definición de funciones es igual que en C, salvo por la característica de que si main no va a recoger argumentos,
                no tenemos por qué ponérselos, a diferencia de C, donde había que ponerlos explícitamente, aunque no se fueran a usar.
                Queda solo comentar que el símbolo << se conoce como operador de inserción, y grosso modo está enviando a
                cout lo que queremos mostrar por pantalla para que lo pinte, en este caso la cadena "Hola mundo".
                El mismo operador << se puede usar varias veces en la misma sentencia, de forma que gracias a esta
                característica podremos concatenar el objeto endl al final, cuyo resultado será imprimir un retorno de línea.</p>

            <p>Un ejemplo de clase que podemos tomar es la clase perro. Cada perro comparte unas características (atributos).
                Su número de patas, el color de su pelaje o su tamaño son algunos de sus atributos. Las funciones que lo hagan ladrar,
                cambiar su comportamiento... esas son las funciones de la clase.</p>

            <p>Son unos métodos especiales que se ejecutan automáticamente al crear un objeto de la clase. En su declaración no se especifica
                el tipo de dato que devuelven, y poseen el mismo nombre que la clase a la que pertenecen. Al igual que otros métodos,
                puede haber varios constructores sobrecargados, aunque no pueden existir constructores virtuales.</p>

            <p>Como característica especial a la hora de implementar un constructor, justo después de la declaración de los parámetros,
                se encuentra lo que se llama "lista de inicializadores". Su objetivo es llamar a los constructores de los atributos
                que conforman el objeto a construir.</p>
        </div>
    </div>
</body>
</html>
